What it is.

A working set of primitives. Not a platform. Not a product. Not a standard. A substrate that higher layers can be built on.

Authority

Ward · Envelope · Warrant

Sovereign protected domains with explicit human-origin acts. Single-use, content-bound, Ed25519-signed warrants that cannot be replayed or repurposed.

Decision

Commit Gate

Deterministic pure function. Same inputs, same decision: ALLOW, REFUSE, ESCALATE, EXPIRE — closed reason-code taxonomy.

Evidence

GEL chain

Hash-chained, signed records of every decision. Offline-verifiable evidence bundles. Counterfactual replay against alternate policy worlds.

Partition

Mesh runtime

ROOT, WITNESS, EDGE roles. Fluidity Tokens for bounded disconnected operation. Real partition tolerance, demonstrated across 40 simulated assets.

Protocol

Hardware adapters

Seven wire-level protocol adapters refuse unauthorized writes before bytes hit the wire. MAVLink, ROS2, OPC-UA, DNP3, Modbus, BACnet, K8s admission.

Verify

Public verifier

A third party with only the warrant + canonical action hash + the issuer's trust anchors can validate offline — without any access to the gate.

The four seams.

The runtime separates four operations that are easy to conflate: evaluation (pure), warrant issuance (signs material), warrant consumption (atomic single-shot), and GEL append (hash-chained). Each is auditable independently.

What's NOT proven.

Listed up front, on purpose. A substrate whose limits are documented is more credible than one whose limits are hidden.

Gap Status Honest framing

The wedge.

What's the smallest, most pointed claim that's true and not addressed by any single comparable?

Warranted execution before consequence: a per-action, signed, content-bound, single-use conveyance, refused at the wire when authority drift is detected, recorded in a hash-chained ledger, verifiable by a third party who holds only the artifact.

Five primitives that don't co-exist in any single comparable product: per-action (not per-session), content-bound (cannot authorize a different action), single-use (consumed at the boundary), refused at the wire (adapter layer rejects drift), verifiable by third party (insurance / audit / regulator).

How it differs from adjacent products.

vs. OPA / Cedar / Casbin

Same evaluation model: subject + action + resource → decision. AristotleOS extends to wire-level actuation, produces signed Warrants, hash-chained evidence, partition-tolerant disconnected operation.

vs. JWT / OAuth / Macaroons

JWTs are reusable bearer tokens. Warrants are single-use + content-bound + nonce-bound + partition-tolerant. Authority you can replay; tokens you can't.

vs. Guardrails (LangChain, NeMo, Bedrock)

Guardrails operate on model output. AristotleOS operates on the action that follows the output — post-output, pre-consequence.

vs. OT cybersecurity (Claroty, Dragos)

Those products detect and monitor. AristotleOS structurally prevents unauthorized writes at the adapter layer — refusal-before-emission.

vs. Sigstore / in-toto / SLSA

Sigstore attests software artifacts. AristotleOS attests agent decisions. Combining them (anchoring GEL roots to Sigstore) is a natural complement.

vs. Cloud IAM

IAM is API access control. AristotleOS extends governance through to wire-level actuation across seven protocols, with cryptographic per-action evidence.
