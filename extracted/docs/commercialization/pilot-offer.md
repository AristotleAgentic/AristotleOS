# 90-Day Disconnected Autonomy Governance Pilot

## Offer

AristotleOS maps one consequential autonomous workflow into a governed execution boundary: Ward, Authority Envelope, Warrant, Commit Gate, refusal states, degraded-authority behavior, evidence, and post-reconnect reconciliation.

## Timeline

| Timeline | Deliverable |
| --- | --- |
| Weeks 1-2 | Authority-to-execution mapping for one autonomous workflow. |
| Weeks 3-4 | Define Ward, Authority Envelope, Warrant logic, refusal states, degraded-authority rules, and evidence requirements. |
| Weeks 5-6 | Integrate one adapter, simulated adapter, or workflow boundary. |
| Weeks 7-8 | Run disconnected/degraded execution scenarios. |
| Weeks 9-10 | Produce GEL evidence bundle and reconciliation report. |
| Weeks 11-12 | Deliver findings, limitations, roadmap, and production integration plan. |

## Buyer Fit

Good pilot candidates have consequential autonomous or semi-autonomous actions, degraded connectivity risk, authority or safety review requirements, and a buyer who needs evidence after incidents or exercises.

## Honest Boundary

The pilot does not replace certified control systems. AristotleOS sits upstream as the runtime governance layer for proposed consequential actions.
